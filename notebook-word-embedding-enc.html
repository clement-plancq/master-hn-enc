def count_words(filepath):
    """
    Compte la frquence de chaque mot du texte donn en argument
    texte au format conll
    """
    words_freq = Counter()
    with open(filepath) as doc:
        for line in doc:
            cols = line.split("\t")
            if len(cols) > 4:
                form = cols[1]
                lemma = cols[2]
                pos = cols[3]
                if pos != "PONCT":
                    words_freq[form] += 1
    return words_freq

def get_words(filepath):
    """
    Retourne les mots d'un fichier conll sous forme de liste
    """
    words = []
    with open(filepath) as doc:
        for line in doc:
            cols = line.split("\t")
            if len(cols) > 4:
                form = cols[1]
                lemma = cols[2]
                pos = cols[3]
                if pos != "PONCT":
                    words.append(cols[1])
    return words

2. Vecteurs de co-occurrences¶

Si on veut conserver l'information lie au contexte d'apparition des mots d'un corpus il est possible de calculer des vecteurs de co-occurrences en comptant le nombre d'occurrences d'un mot pour chacun de ses n voisins.
tablir une matrice de co-occurrence pour un corpus d'un vocabulaire de 54596 mots comme le ntre est difficilement ralisable, a donnerait une matrice de dimension 54596 x 54596.

3. Words embeddings avec Word2vec¶

En 2013, Mikolov et ses co-auteurs ont propos avec Word2Vec une mthode pour vectoriser les mots en prenant en compte leur contexte d'apparition dans des vecteurs de dimensions rduites.

La grande nouveaut de Word2Vec est d'utiliser des prdictions plutt que des comptages.
Pour la mthode Skip-gram, plutt que compter combien de fois un mot w apparat ct de abricot, on va entraner un classifieur pour la question est-ce que w a des chances d'apparatre dans le contexte de abricot ? . Les poids du classifieur seront utiliss dans le vecteur du mot abricot.
L'ide est d'utiliser les contextes d'un mot comme un gold pour la question du dessus.

3.2 Entraîner un modèle¶

Nous allons utiliser le module gensim pour entraner un modle word2vec sur notre corpus de romans de Victor Hugo.

Notre unit de traitement pour les donnes d'entre ce sera la phrase. Nous allons convertir notre corpus en une liste de phrases. Le gros intrt de word2vec et des words embeddings en gnral est de pouvoir partir de donnes brutes, non tiquetes. Nanmoins le travail de prparation du corpus a une grande importance dans le rsultat final :tokenisation, normalisation de la casse et ventuellement traitement des mot composs.

In [29]:
sentences = []
sentence = []

Dans la cellule qui suit nous allons entraner un modle word2vec avec notre corpus organis en phrases.

Le constructeur de la classe Word2Vec peut tre appel avec beaucoup de paramtres : voir la doc ici
Nous allons utiliser les hyper-paramtres suivants :

  • size :la dimension des vecteurs (dfaut: 100)
  • window :la taille en mots de la fentre du contexte
  • sg : l'algo utilis (1 pour skip-gram, 0 pour cbow)
  • iter : nombre d'itrations sur le corpus

workers est le nombre de curs logiques que vous allouez au calcul. Attention ne pas utiliser un nombre de curs trop grand.

def display_words_vectors_pca(model, words):
    word_vectors = np.array([model[w] for w in words])

        plt.text(x+0.05, y+0.05, word)
